01 · Introducción

El Problema y el Contexto

🎯 Problema Identificado

  • La incompatibilidad de formatos multimedia es un desafío diario para estudiantes, docentes y profesionales tecnológicos.
  • Formatos como MKV, AVI, MP4, WebM, MOV y FLV no son universalmente compatibles.
  • Las soluciones existentes carecen de control sobre el hardware o son demasiado complejas para el usuario común.
  • No existe integración de auditoría ni gestión de usuarios en herramientas gratuitas del mercado.

💡 Solución Propuesta

VIDEOFORGE es una aplicación de escritorio Windows en C# (.NET Framework) que:

  • Usa el motor FFmpeg para transcodificación de alta calidad.
  • Implementa concurrencia controlada con SemaphoreSlim.
  • Soporta GPU: NVIDIA NVENC, AMD AMF, Intel QSV.
  • Gestiona usuarios y auditoría con MySQL + SHA-256.

03 · Características Técnicas

Features Backend Implementadas

🎮

Aceleración GPU Multi-Fabricante

h264_nvenc (NVIDIA), h264_amf (AMD), h264_qsv (Intel). Detección automática por nombre de GPU.

🔀

Conversión Paralela con SemaphoreSlim

SemaphoreSlim(maxHilos) limita las conversiones simultáneas. Task.WhenAll() espera la finalización de todas.

⏸

Pausa/Reanudación a Nivel SO

NtSuspendProcess congela el proceso FFmpeg a nivel kernel. NtResumeProcess lo reanuda sin pérdida de estado.

📊

Progreso en Tiempo Real

Regex sobre stderr de FFmpeg extrae Duration: y time=. Evento ProgresoReportado actualiza la UI.

🔒

Thread-Safety en UI

Todos los updates al DataGridView se realizan via this.Invoke(MethodInvoker) garantizando ejecución en el hilo principal.

🚫

Cancelación y Limpieza

CancellationTokenSource coordina la cancelación. Al cancelar: Kill() y File.Delete() para evitar archivos corruptos.

📋 Entregable E-04: Requisitos No Funcionales (RNF-01 a RNF-08)

Definí los 8 RNF con categorías (Rendimiento, Usabilidad, Seguridad, Confiabilidad, Escalabilidad, Compatibilidad, Mantenibilidad, Portabilidad), métricas específicas y plan de verificación para cada uno. Estos criterios definen la calidad del software, no solo su funcionalidad.

⚡ Plan de Pruebas de Rendimiento

Diseñé el plan de prueba: conversión de archivos de 1 GB y 4 GB midiendo tiempo en CPU (libx264) y GPU (h264_nvenc, h264_amf, h264_qsv). Prueba de escalabilidad con SemaphoreSlim configurado en 1, 2, 4 y 8 hilos con 20 archivos simultáneos. Meta: <2 min/GB CPU y <45 seg/GB GPU.

🔐 Verificación de Seguridad y Portabilidad

Definí los procedimientos de verificación de RNF-03 (Seguridad): comprobar en MySQL que las contraseñas son hashes SHA-256 de 64 caracteres. Y RNF-08 (Portabilidad): cambiar el servidor MySQL en db_config.ini y verificar reconexión exitosa sin recompilar.
